按照变更规划的任务清单逐项实现代码。

---

**输入**: 变更名称（可选）。

**步骤**

1. **选择变更**

   如果提供了名称，直接使用。否则从对话上下文推断。如果只有一个活跃变更，自动选择。如果模糊，列出 `.claude/har/changes/`（排除 archive/）让用户选择。

2. **读取上下文**

   读取变更目录下的所有制品：
   - `proposal.md` — 动机和范围
   - `design.md` — 技术方案
   - `specs/` — 需求规格
   - `tasks.md` — 任务清单

3. **显示进度**

   解析 tasks.md 的 checkbox 状态，展示已完成/待完成任务数。

4. **逐项实现**

   对每个 `- [ ]` 任务：
   - 展示任务编号和描述
   - 实现代码变更
   - 将 `- [ ]` 改为 `- [x]`
   - 继续下一个

5. **暂停条件**

   - 任务不清晰 → 请求澄清
   - 发现设计问题 → 建议 `/har:update`
   - 遇到错误/阻塞 → 报告并等待
   - 用户中断

6. **完成总结**

   显示本次完成的任务和总体进度。全部完成时建议 `/har:archive`。

**护栏**
- 持续执行直到完成或阻塞
- 实现前先读取所有制品
- 任务模糊时暂停，不猜测
- 发现设计问题时暂停
- 代码变更小且聚焦
- 立即更新 checkbox
