---

修订变更的已有规划制品并保持它们之间的一致性。只编辑规划文件，绝不编辑代码。

---

**输入**: 变更名称（可选）和修订描述。

**步骤**

1. **选择变更**

   如果未提供名称，列出 `.claude/har/changes/`（排除 archive/）中的变更让用户选择。不自动选择。

2. **读取已有制品**

   读取变更目录下的所有已有制品文件。

3. **理解修订请求**

   - 用户指定了具体修订 → 以此为起始编辑
   - 用户只说"更新" → 进行一致性审查：检查矛盾、缺口、重复

4. **读取并协调**

   - 应用请求的编辑
   - **双向检查**每个其他已有制品：
     - 修改 proposal → 检查 design、specs、tasks 是否需更新
     - 修改 design → 检查 proposal、specs、tasks 是否需更新
     - 修改 tasks → 检查 design、proposal 是否需更新
   - 仅修改已有文件，不创建新制品
   - 记录缺失制品，建议用 `/har:plan`

5. **确认后逐个写入**

   展示每个拟修订内容和理由，用户确认后再写入。用户拒绝的修订不写入。

6. **提示下一步**

   - 缺失制品 → `/har:plan`
   - 规划已修订但代码未更新 → `/har:apply`
   - 全部完成 → `/har:archive`

**护栏**
- 只编辑规划制品，不编辑代码
- 仅修改已有文件
- 每个编辑确认后再写入
- 变更意图改变时建议新建变更
